# OpenSpec Change Checklist

## Official commands

- Check CLI: `openspec --help`
- Bootstrap current repo carefully: `openspec init`
- Review generated governance and spec files before merging them into an existing repo

## Baseline

- What works today
- What is broken or insufficient
- What contracts must not break

## Change surface

- Runtime
- Storage / data model
- APIs / prompts / commands
- Docs / operator workflow
- Tests / monitoring

## Safety gates

- backward compatibility stated
- migration path stated
- rollback path stated
- verification commands listed
- operator impact called out
