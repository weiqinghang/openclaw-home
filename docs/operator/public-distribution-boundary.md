# 公共发行模板与作者实例边界

目标：让这个仓库在不影响作者当前可用性的前提下，逐步具备更稳定的复用能力。

## 先记住一句话

- `openclaw.json` 是**作者本人当前运行中的实例配置**
- `openclaw.template.json` 是**公共发行模板**
- `secrets.local.example.json` 是**公共 secrets 示例**

不要把 `openclaw.json` 当成新机器默认起点。

## 明确支持版本

当前这个仓库**明确验证过**的 OpenClaw 版本是：

- `2026.3.13`

说明：

1. OpenClaw 上游迭代很快，CLI、gateway、ACP、插件布局都可能变化
2. 因此作者不会承诺“任意新版本都自动兼容本仓库”
3. 超出已验证版本后，clone 用户应优先让自己的 Coding Agent先做本地兼容检查
4. 如果不希望自己适配，可以等待作者把仓库兼容层补齐后再升级

推荐态度：

- **已验证版本**：优先直接使用
- **未验证版本**：允许尝试，但默认视为可能需要额外适配

## 为什么要分层

这个仓库同时承载两类东西：

1. 作者当前正在跑的 OpenClaw 实例
2. 其他人 clone 后希望复用的公共资产

如果不分层，常见问题是：

- 新人误抄作者机器路径
- 把运行中的项目、账号、测试条目当成公共默认配置
- 为了“开箱即用”去改坏作者本人正在使用的入口

因此本仓库默认采取：

- **作者实例优先保持可用**
- **公共发行能力通过模板、示例、文档逐步补齐**

## 文件职责

### 1. `openclaw.json`

用途：

- 作者本人当前实际在用的配置
- 允许包含作者机器路径、当前项目、当前实验性接入

约束：

- 不把它当公共模板
- 不要求它对所有 clone 用户直接可运行
- 任何变更都优先保证作者当前链路不被破坏

### 2. `openclaw.template.json`

用途：

- 给新环境复制生成本地 `openclaw.json`
- 表达仓库当前认可的“最小公共发行版”

约束：

- 只放公共起步所需能力
- 不混入作者个人实验项
- 路径、账号、用户 ID 都必须是占位符

### 3. `secrets.local.example.json`

用途：

- 给新环境复制生成本地 `secrets.local.json`
- 与 `openclaw.template.json` 保持一致

约束：

- 只覆盖模板里实际启用的账号
- 不承载作者私有或可选扩展项的必填要求

## 当前公共发行版范围

当前默认公共发行版只覆盖：

- `wukong`
- `taibai`
- `guanyin`
- `guichengxiang`

以下内容目前视为**作者实例或可选扩展**，不应默认要求 clone 用户立即具备：

- `laojun`
- 项目级 `test-project`
- 作者机器专属路径
- 本机补丁态与实验性运行项

## 改造原则

后续若继续推进“更开箱即用”，遵循以下顺序：

1. 先补模板、示例、文档
2. 再补兼容脚本
3. 最后才考虑收口作者实例与公共发行版的差异

禁止反过来做：

- 不先删作者当前配置
- 不先把作者实例强行改成公共模板
- 不为了公共发行，直接破坏作者当前命令入口

## 什么时候改哪份文件

如果你在做的是：

- **修作者当前实例问题** -> 改 `openclaw.json`
- **提升 clone 用户起步体验** -> 改 `openclaw.template.json` / `secrets.local.example.json` / `README.md`
- **同时影响两边** -> 先保作者实例，再同步模板

## 最小验收标准

边界分层要同时满足两条：

1. 作者当前命令仍可用
2. 新用户能从模板开始完成最小接入

少任意一条，都不算改造完成。
