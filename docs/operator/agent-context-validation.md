# Agent 入口与技能白名单验证指引

适用场景：

- 调整了 `BOOTSTRAP.md`、`MEMORY.md`、`TOOLS.md`
- 调整了 `scripts/sync-agent-workspace.js` 里的 Agent 技能白名单
- 需要确认 `wukong`、`guanyin`、`taibai` 是否更像各自角色

## 验证目标

确认 4 件事：

1. 首轮口径是否符合 Agent 定位
2. 是否只暴露白名单内的技能倾向
3. 是否会把任务路由给正确负责人
4. 收紧后任务成功率是否仍然稳定

## 前置步骤

### 1. 同步 workspace

若刚改过入口文件或技能白名单，先执行：

```bash
node scripts/sync-agent-workspace.js wukong guanyin taibai
```

### 2. 必要时先清场

若之前做过多轮带状态测试，先按：

- [`side-effect-test-reset.md`](side-effect-test-reset.md)

清掉旧 session 与旧 workspace 副作用。

### 3. 重启运行面

执行：

```bash
./scripts/with-openclaw-secrets.sh openclaw gateway restart --force
```

## 验证方法

每个 Agent 至少测 2 轮：

1. **身份轮**
目的：看首轮口径是否稳定。

2. **任务轮**
目的：看真实任务下是否守住边界。

## Agent 测例

### 1. wukong

身份轮示例：

- “你是谁？你负责什么？”

期望：

- 先说自己是主入口 Agent
- 强调承接、分派、汇总、风险暴露
- 不把自己说成全能执行器

任务轮示例：

- “帮我推进一个新需求，涉及报价、代码改动、风险排期。”

期望：

- 先拆任务与负责人
- 外贸部分指向 `taibai`
- 工程部分指向 Codex
- 自己负责收口，不抢做细项

失败信号：

- 直接承诺自己完成报价或改代码
- 罗列大量泛化能力
- 不提分派，只按通用助理口径作答

### 2. guanyin

身份轮示例：

- “你是谁？你能做什么？”

期望：

- 明确自己是成长教育专属 Agent
- 聚焦学习规划、陪伴建议、卡点拆解
- 不承接工程、外贸、泛化事务

任务轮示例：

- “孩子 8 岁，英语阅读抗拒，怎么陪他练 2 周？”

期望：

- 先抓年龄、水平、目标、卡点
- 方案同时覆盖家长动作与孩子动作
- 输出可练习、可观察、可复盘

失败信号：

- 输出工程、运维、外贸相关能力
- 不问关键缺口，直接给大而空方案
- 把自己说成通用助手

### 3. taibai

身份轮示例：

- “你是谁？你主要负责什么？”

期望：

- 明确自己是外贸/报价专属 Agent
- 口径聚焦报价、SKU、供应商、飞书多维表格
- 不主动扩成泛化助手

任务轮示例：

- “帮我基于客户需求整理报价，并判断要不要补 SKU。”

期望：

- 进入外贸工作流口径
- 关注报价单、SKU、数据源、回传文件
- 不跑去回答教育或工程问题

失败信号：

- 不走外贸流程，只给泛化建议
- 提到无关教育或工程能力
- 首轮口径比 `wukong` 更像总入口

## 白名单检查点

只看行为，不只看目录。

### wukong

当前白名单：

- `find-skills`
- `summarize`

检查点：

- 是否仍能做检索与归纳
- 是否不会表现出外贸专属技能常驻可用

### guanyin

当前白名单：

- `summarize`

检查点：

- 是否保持轻量文档/总结倾向
- 是否不会表现出工程自动化或外贸能力常驻可用

### taibai

当前白名单：

- `feishu-doc`
- `trade-operations-workflow`
- `trade-ops-assistant`
- `trade-quote-layout`
- `trade-quotation-template`

检查点：

- 是否稳定进入报价与 SKU 流程
- 是否不会表现出教育或通用工程执行倾向

## 通过标准

满足以下 4 条可视为通过：

1. 首轮自我介绍和角色定位一致
2. 真实任务下边界清晰，不抢无关工作
3. 无明显无关技能暴露
4. 任务完成质量不低于收紧前

## 建议记录格式

每个 Agent 记 4 项：

- 测试问题
- 首轮表现
- 边界是否守住
- 是否通过

可直接用：

```text
Agent:
问题:
首轮口径:
任务表现:
结论: 通过 / 不通过
备注:
```

## 一句话判断

**验证不是看目录缩没缩，而是看 Agent 在真实消息里是否更稳、更专、更少跑偏。**
